

## Code Path: Deleting a Claude `.jsonl` File Before `claude_consult`

### The Core Check

The critical function is `sessionFileExists()` in `src/claude.ts:104–122`. It looks for:

```typescript
path.join(home, ".claude", "sessions", `${claudeSessionId}.jsonl`)
```

If that file is deleted, the check returns **false**.

- The old orphaned session record remains in the DB, marked `orphaned`

### What Happens to the Deleted Session

| Aspect | Result |
|---|---|
| Session record in DB | Marked `orphaned` (status change), not deleted |
| History/context | **Gone** — new session has no conversation history |
| Next auto-route match | Session still in candidate pool but will be re-checked and orphaned again if still missing |
| Orphan recovery flag | `wasOrphanRecovery: true` in the consult result's `routing` object |
