

The path when a `.jsonl` file is deleted before `claude_consult`:

---

**1. `consultService.consult()`** (`src/consult.ts:52`) — acquires `cost:` lock, calls `resolveRoute()`

**2. `resolveRoute()`** (`src/consult.ts:249`) — two cases:

**If `session_id` explicitly provided:**
- Session found in SQLite; project mismatch checked (`src/consult.ts:251-257`)
- `sessionFileExists(claude_session_id)` called (`src/claude.ts:104-122`) — searches `~/.claude/sessions/{id}.json`, `sessions/{id}.jsonl`, and deep-scan in `~/.claude/projects`
- **File missing** → `db.markSessionOrphaned(session.id)` (`src/db.ts:518-530`): status set to `orphaned`, `orphan_recovery` event logged
- Returns `RouteResolution` with `selectedSession = null`, `bootstrapContext = buildBootstrapContext(oldView, "orphaned")`, `wasOrphanRecovery = true`

**If `session_id` null (auto-route):**
- `listMatchCandidates` skips orphaned sessions silently
- `sessionFileExists` checked on each candidate before considering reuse (`src/consult.ts:300-301`)
- Orphaned sessions excluded from scoring

**3. `invokeWithRouteLock()`** (`src/consult.ts:97-115`) — since `selectedSession = null`, skips route lock, goes straight to `invokeClaude()`

**4. `invokeClaude()`** (`src/consult.ts:117`):
- Bootstrap context (topic, summary, decisions, files, tags, aliases) injected into prompt via `buildSessionContext()` or `buildBootstrapContext()`
- `claude.runPrompt(prompt)` called **without `--resume`** — new Claude session created
- New `claude_session_id` returned and stored in SQLite
- Event `new_session` logged with `was_orphan_recovery = true`

---

**Outcome**: Orphaned session stays in SQLite as historical record; new session created; compact registry context transferred so no architectural memory is lost. The `was_orphan_recovery = true` flag in `session_events` provides full observability for the routing decision.
