

Three concrete reasons, all grounded in the code:

**Purpose fit** (`src/cluster.ts:467-495`): The verifier prompt says *"Do not infer. Do not use tools. Return JSON only."* The verifier judges pre-collected evidence — running `agent` would unleash `Agent`/`Glob`/`Read` exploration that contradicts the evaluation task.

**Cost control**: `agent` enables full tool access, which is exactly what the cache layer exists to avoid. The measured baseline from `docs/CLUSTER_CACHE_SPEC.md:17-20`: `bare` consult uses ~1,170 tokens vs. 56,051 for `Agent`+`Glob`+`Read`. Running the verifier in `agent` would destroy that ratio.

**Explicit guard** (`src/tools.ts:526-528`): The LLM verifier path throws before it ever calls Claude if the selected profile is `agent`:
```ts
if (profileSelection.selected === "agent") {
  throw new Error("LLM verifier cannot use agent profile");
}
```

The default is `focused` (`src/tools.ts:116`) — `bare` is only tried if explicitly requested. The downgrades are deterministic: `bare → focused → throws` (never `agent`).
