

Three reasons, all grounded in the code:

**Purpose fit** — The verifier prompt at `src/cluster.ts:467-495` explicitly says: *"Do not infer. Do not use tools. Return JSON only."* The verifier judges pre-collected evidence; running `agent` would unleash `Agent`/`Glob`/`Read` exploration that contradicts the evaluation task.

**Cost control** — `agent` enables full tool access. The cluster cache layer's measured baseline (`docs/CLUSTER_CACHE_SPEC.md:17-20`): `bare` consult uses ~1,170 tokens vs. 56,051 for `Agent`+`Glob`+`Read`. Running the verifier in `agent` would destroy the cost ratio this entire layer is designed to preserve.

**Explicit guard** (`src/tools.ts:526-528`) — The LLM verifier path throws before ever calling Claude if the selected profile is `agent`:
```ts
if (profileSelection.selected === "agent") {
  throw new Error("LLM verifier cannot use agent profile");
}
```

The default is `focused`, with `bare` tried only if explicitly requested. The downgrade chain is deterministic: `bare → focused → throws`, **never `agent`**.
